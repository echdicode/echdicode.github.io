Lý do tôi biết đến CLEAN CODE

Sau khi đọc review sách Clean Code: A Handbook of Agile Software Craftsmanship của tác giả Robert C.Martin Blog tại blog toidicodedao mình bắt cũng bắt đầu tìm hiểu và đọc cuốn sách này và học các để biết code nào là “xịn” code nào là “rởm”.Và trong series này mình sẽ chia sẻ tóm tắt những gì mình đọc và học được từ cuốn sách này .

CLEAN CODE ?

Chương đầu tiên của cuốn sách có tiêu đề là Clean Code chương này sẽ giúp chúng ta định nghĩ về code sạch và các tác hại của việc viết code “rởm”.

Tác hại của việc viết code “rởm” chắc chắn người lập trình viên nào cũng sẽ trải qua thôi. Bản thân mình đây mới là sinh viên học nhập môn ngôn ngữ lập trình viết vài dòng code giải vài bài toán đơn giản cũng bị dính trưởng huống gì mấy cái project to tổ bố. Nhiều lúc lâu lâu mình đọc lại code mình hay code của mấy bạn khác còn mãi mới hình dung được là đang viết cái gì đó là một tác hại của việc viết code “rởm”.Trong cuốn sách có một đoạn:

Nếu bạn là một lập trình viên đã làm việc trong 2 hoặc 3 năm, rất có thể bạn đã bị mớ code lộn xộn của người khác kéo bạn lùi lại. Nếu bạn đã là một lập trình viên lâu hơn 3 năm, rất có thể bạn đã tự làm chậm sự phát triển của bản thân bằng đống code do bạn tạo ra.

Đoạn này cho chúng ta rất nhiều suy ngẫm về việc viết code ẩu không chỉ kéo mình xuống mà còn kéo cả một tập thể đi xuống.

Vậy thì viết code sạch là cái gì ? ở đây mình sẽ trích một số định nghĩa về code sạch của một số lập trình viên nổi tiếng thế giới.

Tôi thích code của tôi trông thanh lịch và hiệu quả. Sự logic nên được thể hiện rõ ràng để làm cho các lỗi khó lẩn trốn, sự phụ thuộc được giảm thiểu để dễ bảo trì, các lỗi được xử lý bằng các chiến lược rõ ràng, và hiệu năng thì gần như tối ưu để không lôi kéo người khác tạo nên code rối bằng những cách tối ưu hóa tạm bợ. Code sạch sẽ tạo nên những điều tuyệt vời.
cre :JARNE STROUSTRUP – CHA ĐẺ CỦA NGÔN NGỮ C++, VÀ LÀ TÁC GIẢ CỦA QUYỂN THE C++ PROGRAMMING LANGUAGE

Code sạch đơn giản và rõ ràng. Đọc nó giống như việc bạn đọc một đoạn văn xuôi. Code sạch sẽ thể hiện rõ ràng ý đồ của lập trình viên, đồng thời mô tả rõ sự trừu tượng và các dòng điều khiển đơn giản.
cre: GRADY BOOCH, TÁC GIẢ QUYỂN OBJECT ORIENTED ANALYSIS AND DESIGN WITH APPLICATIONS

Bạn biết bạn đang làm việc cùng code sạch là khi việc đọc code hóa ra tuyệt vời hơn những gì bạn mong đợi. Bạn có thể gọi nó là code đẹp khi những dòng code đó trông giống như cách mà bạn trình bày và giải quyết vấn đề
cre: Ward Cunningham, người tạo ra Wiki

Tóm lại code sạch là code dễ đọc, dễ hiểu, dễ sửa chữa và bảo trì nó làm cho người đọc cảm thấy yomost. Nói thì dễ thế thôi để mà có thể clean code thì chúng ta cần phải có thói quen luyện tập và tình thần kiên trì sắt đá.Clean code là một nghệ thuật còn người viết clean code là một nghệ sĩ.
